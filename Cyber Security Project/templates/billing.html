How it works

The billing system stores cashiers in a SQLite users table. Two login functions live in modules/billing_login.py:

  • Vulnerable: builds SQL by string concatenation.
  • Secure: uses a parameterized query (? placeholders).

Try this payload in the Vulnerable tab to bypass login: username = admin' OR '1'='1 -- , password = anything. Then switch to Secure and try the same payload - it will fail, because the injection is treated as a literal string.
